# AgentSAST — Claude Code guide

SAST agent in a multi-agent API scanner. Go/Gin/GORM + MySQL + MinIO. It pulls a
project's raw upload from MinIO, runs an LLM (the **Codex CLI**) over it via an
editable **skill**, and writes back an enriched OpenAPI + a DAST-oriented report.
Only `./AgentSAST` is ours — keep changes here to avoid conflicts with teammates
(`Manager`, `AgentDAST`).

Module `agentsast`, Go 1.26. Full details in `README.md`; demo in `mock/README.md`.

## Run & test (Docker-first)

Everything runs in Docker — **do not** use the host `codex` CLI or a local
`.codexhome`. Use `docker-compose` (v1, hyphenated), **not** `docker compose`.

```bash
go build ./... && go vet ./... && gofmt -l .   # fast local check (do this often)
docker-compose up -d mysql minio minio-init
docker-compose up -d --build agentsast         # rebuild after any Go change
curl -s localhost:8081/api/sast/health         # expect ping.llm = "ok"
./mock/minio.sh                                 # seed demo_project to MinIO
# then open mock/manager.html, or curl /run (see mock/README.md)
```

A real scan costs the user's LLM credits and takes ~1–2 min. **Don't run real
scans casually** — verify with `go build`/`vet` first; only do an end-to-end run
when behavior actually needs confirming. There is no mock mode (removed).

## Data flow

`POST /run {project_id (UUIDv4), swagger_base_url}` → job goroutine:
1. download `<project_id>/raw/` → `<workRoot>/<id>/raw/`
2. `codex exec` in that workspace → skill extracts archives, analyzes code, writes
   `sast/openapi.yaml`, `sast/report.md`, `sast/base_url.txt`
3. upload `openapi.yaml` + `report.md` to `<project_id>/sast/*` (base_url.txt stays internal)
4. persist `result_*` + verified `result_swagger_base_url`

Status: `new` → `process` → `done`/`failed`/`canceled`. `last_message` = latest
Codex emit (or error); `last_update` = heartbeat (bumped per event).

## Where to change behavior

- **SAST logic / prompts** → `skills/sast/SKILL.md` + `skills/sast/references/*.md`
  (idor, sqli, xss, payment). This is the main lever — *not* Go code. The skill is
  installed into `CODEX_HOME` at startup by `internal/service/codex/setup.go`.
- **Codex invocation / flags** → `internal/service/codex/runner.go`; the init
  prompt → `internal/service/codex/codex.go`.
- **Orchestration** (download→codex→upload, status writes) → `internal/service/job/job.go`.
- **HTTP routes** → `internal/handler/route_*.go` (+ `router.go`, `Deps`).
- **DB model** → `internal/db/model/sast.go`; **config** → `config.yaml` + `internal/config`.

## Gotchas (learned the hard way)

- **`wire_api = "responses"`** is required in the generated codex `config.toml`;
  codex 0.139+ removed `"chat"` (hard config-load error).
- **Do NOT use `codex exec --output-schema`** with this proxy/model
  (minimax-m2.5 / responses API): structured output short-circuits the agentic
  loop — it returns JSON immediately, runs zero shell commands, writes no files.
  Instead the skill writes a sidecar `sast/base_url.txt` that `runner.go` reads
  (avoids YAML parsing too — a strict full-doc YAML parse breaks on LLM stray colons).
- **Dedicated `CODEX_HOME`** (container `/root/.codex`) — never the user's real
  `~/.codex` (would clobber their profile/trust). Generated by `setup.go`.
- **LLM config is from env** (`LLM_BASE_URL/LLM_API_KEY/LLM_MODEL/LLM_EFFORT`),
  interpolated into `config.yaml`. The app auto-loads `../.env` then `./.env`.
- `mock/minio.sh` uploads via `mc pipe` over **stdin** (Docker Desktop doesn't
  share macOS tmp paths, so bind-mounting the zip fails).
- `mock/demo_project` is a **separate Go module** (nested `go.mod`) so its
  deliberately-vulnerable code is excluded from `go build ./...`.
- Changing the `sast` schema: GORM AutoMigrate won't shrink existing columns —
  `DROP TABLE sast;` for a clean schema during dev.
- A stale `OPENAI_API_KEY` in the shell can cause 401s (irrelevant once the custom
  `agentsast` provider + `LLM_API_KEY` is used).
